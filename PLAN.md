# CHEERDMOTO News 自动发布执行计划

## 当前架构

- Next.js App Router、Vercel 和 Vercel Blob 持久化。
- 新闻正文与元数据保存于后台内容库；新闻审计、任务日志和 Sitemap 变更事件分别持久化保存。
- 前台路径为 `/news`、`/news/[slug]`，产品路径为 `/products/[slug]`。

## 发布流程

1. 从 `NEWS_RSS_FEEDS` 拉取公开 RSS/Atom，不抓取受限页面或文章全文。
2. 只接受来源白名单、允许语言、明确原始发布时间且在 `NEWS_LOOKBACK_HOURS` 内的候选。
3. 以来源规范 URL、规范标题和 SHA-256 指纹执行连续 `NEWS_DEDUP_DAYS` 去重。
4. 根据现有公开产品名称、分类和 slug 计算相关性；没有真实产品关联或低于阈值的候选不发布。
5. 生成事实摘要、明确来源、独立分析和真实产品链接，使用 CHEERDMOTO 自有产品图作语境插图。
6. 持久化文章、产品关联、审计日志和任务日志；随后刷新新闻与相关产品路径并记录 Sitemap 变更。

## 调度与质量控制

- Vercel 每天分四个时段调用同一发布接口；每次最多发布一篇，日上限四篇。
- RSS 拉取有有限重试；来源故障、过期、重复、低相关性和未配置来源都会写入审计，而非静默发布。
- `NEWS_AUTO_PUBLISH=false` 时只记录候选，供人工审核。
- 生产必须配置 `CRON_SECRET`，未配置时 cron 接口拒绝访问。

## 已知外部依赖

- 可信 RSS 来源与来源白名单通过 Vercel 环境变量管理。
- 可选告警邮箱使用现有 SMTP 配置；当前失败状态在中文后台“News 自动化”中可见。
- 新闻来源、产品事实和图片版权必须持续人工审阅；系统不伪造新闻事实或产品能力。
