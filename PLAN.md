# CHEERDMOTO B2C 管理后台执行计划

## 当前状态

项目已经具备前台网站、中文后台、Vercel 部署、Blob 持久化、SMTP 表单通知、Google Search Console 服务账号配置、基础订单和访问分析能力。本轮根据《B2C零售网站管理后台开发总指令》补齐后台模块入口和核心数据页面。

## 已完成模块

- 数据概览：订单、收入、访客、转化、商品、内容、线索。
- 商品管理：商品、分类、商品属性与规格、库存、媒体库。
- 订单管理：订单列表、订单详情、支付状态、物流和退款动作。
- 支付与退款：Oceanpayment 配置状态、支付回调、退款记录、预授权记录。
- 发货与物流：物流商、追踪号、发货状态、客户可见备注。
- 退换货管理：退款记录和退回状态聚合。
- 客户管理：客户/线索、客户分组、表单记录。
- 购物车与弃购：结账事件和疑似弃购会话。
- 优惠与促销：当前优惠码使用情况。
- 评价管理：后台入口和待接入检查，不伪造评价数据。
- 内容管理：博客、新闻、SEO 数据。
- 数据同步：持久化存储、GSC、Cron 同步状态。
- 用户与权限：当前单管理员状态和 RBAC 角色模板。
- 操作日志：登录成功/失败审计记录。

## 数据架构

- 商品、分类、媒体、内容、设置：`admin-store.json`
- 订单：`orders.jsonl`
- 访问事件：`analytics-events.jsonl`
- 邮件日志：`email-logs.jsonl`
- 支付回调：`payment-notifications.jsonl`
- 退款：`refunds.jsonl`
- 物流：`shipments.jsonl`
- 预授权：`payment-authorizations.jsonl`
- 后台审计：`admin-audit.jsonl`
- Google SEO 快照：`google-seo-snapshot.json`

当前线上持久化优先使用 Vercel Blob。后续如果订单量增加，建议迁移到 PostgreSQL 并保留 Blob 作为媒体和导入导出文件存储。

## 支付架构

- 当前支付适配：Oceanpayment embedded gateway。
- 支持支付创建接口与回调通知接口。
- 回调需要签名校验和幂等记录。
- 后台不保存银行卡号、CVV。
- 退款记录已入库，退款审批流和多角色权限需要数据库 RBAC 后继续强化。

## 订单状态机

- 订单状态：待付款、已付款、处理中、已发货、已送达、已完成、已取消、已退款、部分退款、失败。
- 支付状态：未提交、处理中、成功、失败、已退款、部分退款。
- 物流状态：未发货、已发货、运输中、已送达、已退回。
- 售后状态与支付、物流分开显示，避免互相覆盖。

## 库存策略

- 当前库存来自后台商品库字段 `stock`。
- 已显示低库存和缺货商品。
- 后续需要新增库存流水、锁定库存、释放库存、仓库维度和并发扣减。

## 安全与权限

- 当前为单管理员环境变量登录。
- 已具备登录限流、HttpOnly 会话 Cookie、登录审计。
- 建议生产环境启用 `ADMIN_PASSWORD_HASH`。
- 多管理员、角色权限、会话管理、2FA、首次登录强制改密仍需要数据库表支持。

## 测试与部署

- 每次上线前执行 `npm run build`。
- 部署目标为 Vercel production。
- 关键路径需要验证：首页、后台登录、后台新增页面、订单、支付、SEO 同步、表单邮件。

## 仍需外部确认/开通

- Oceanpayment 正式商户凭据和回调域名白名单。
- Google Search Console 数据权限是否持续有效。
- 邮箱 SMTP 授权是否长期稳定。
- 退换货政策、退款审批规则、评价展示策略。
- 多管理员账号和角色权限的正式数据库方案。
- 税费、发票、地区禁售、物流承运商接口。
